Welcome to the API

Create Card

Method: POST

Triggers TTS generation for the front and back of card

This mostly won't work properly until S3 Bucket is properly implemented

Get Card

Method: GET

Gets TTS file

This mostly won't work properly until S3 Bucket is properly implemented.

To use, add text to the end of the endpoint

Make User

Method: POST

Creates a user in the DB. If user already exists, it will stop.

Pass in JSON of the following form.

{
'UserID': str // probably use whatever token is given by Google
}

Make Card Set

Method: POST

Creates a CardSet owned by a user in the DB.

Pass in JSON of the following form.

{
'UserID': str,
'Name': str
}

Returns: The ID of the Card Set in the database.

Delete Card Set

Method: POST

Deletes a CardSet from the DB.

Pass in JSON of the following form.

{
'UserID': str,
'CardSetID': str
}

Add Cards

Method: POST

Adds cards to a CardSet in the DB.

Pass in JSON of the following form.

{
'UserID': str,
'CardSetID': str,
'cards': [
['front', 'back']
]
}

Delete Card

Method: POST

Deletes a card from CardSet in the DB.

Pass in JSON of the following form.

{
'UserID': str,
'CardSetID': str,
'CardIdx': int // Use zero-indexing for this
}

Update Card

Method: POST

Updates a card in a CardSet.

Pass in JSON of the following form.

{
'UserID': str
'CardSetID': str
'CardIdx': int // Use zero-indexing for this
'NewCard': [ ['front', 'back']
]
}

Get Owned Sets

Method: GET

Gets the Set IDs and Names of a user.

Put User ID at the end of the endpoint.

Returns: JSON in the following form:

{
'SetID1': 'SetName1',
'SetID2': 'SetName2',
...
}

Get Cards in Set

Method: GET

Retrieves the cards in a specific set.

Put Set ID at the end of the endpoint.

Returns: JSON in the following form:

{'set: [
['front1', 'back1'],
['front2', 'back2'],
... }
